REST API

REST API beskriver en server der er state-less, så den svarer på hver forespørgsel på samme måde.

REST API, tager imod information defineret på endpoints, så http protokolen bliver direkte oversat til aktivering af koden.

Endpoints defineres med forskellige http metoder, som GET, DELETE, PUT, PATCH og POST, der fortæller hvad man gerne vil opnå på det endpoint. Information kan tages på forskellige måder:

  • Body: kan sendes med ikke GET-metoder, og kan indeholde en stor mængde data
  • Path: Første dele efter domainet og sat som /[word]/[word]
  • Parameters: efter path og defineret med ’?’[key]=[value]& [key]=[value]
